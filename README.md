Description
-----------
This project is a lightweight, client-first attendance tracker for classrooms and small institutions. It focuses on fast attendance entry (manual or QR), reliable data backup/restore, easy exports, and a neat responsive interface so teachers can record attendance with minimal friction.

Features
--------
- Manual attendance marking and QR-code scanning for fast entry.  
- Backup and restore local data (JSON/IndexedDB or file).  
- Export attendance as CSV and PDF for record keeping.  
- Secure lock: PIN/password to restrict access to the app.  
- Admin/Teacher views for class and student management.  
- Search, filters, and attendance history.  
- Responsive, modern UI with good accessibility and design.  
- Completely free to use.

Built with
----------
- HTML5, CSS3 (Flexbox/Grid), and vanilla JavaScript.  
- Optional libraries: jsQR or ZXing for QR scanning, jsPDF for PDF export.  
- Local storage options: IndexedDB or downloadable JSON backups.

Configuration
-------------
- No DB required for basic usage; data stored in browser storage.  
- To enable persistent server storage or multi-user support, connect your preferred backend and update API endpoints in config.js.  
- Customize app settings (default PIN, export options) in config.js or settings panel.  
- For QR scanning, grant camera permission in the browser and ensure the app is served over HTTPS (or localhost).

Usage
-----
- Admin: create classes, import or add students, set default settings, and manage backups.  
- Teacher: open a class session, mark present manually or scan student QR codes, then save the session.  
- Student: view their own attendance (if enabled) and download personal reports.  
- Use search and filters to locate records by date, class, or student.

Backup & Restore
----------------
- Backup: export the current attendance database as a JSON file for safekeeping.  
- Restore: import a previously saved JSON backup to recover data.  
- Optionally, automatic periodic backups can be enabled (see settings) to download dated backup files.

Exporting Reports
-----------------
- CSV export for spreadsheets and administrative records.  
- PDF export for printable reports (uses jsPDF or similar).  
- Exports include metadata (class name, date range, generated by) and can be filtered before export.

Security / Lock
---------------
- App-level lock: set a numeric PIN or password to restrict access to the interface.  
- Lock protects sensitive actions like viewing full records, exporting, deleting data, and restoring backups.  
- For stronger security, host the app behind an authenticated backend and use HTTPS.
